ภายใต้นโยบาย Cloud First: การหาทางออกที่เหมาะสมในการปรับลดระยะเวลาเช่าเหลือ 12 เดือน

สรุปผลการวิเคราะห์

ข้อสรุปหลัก

หากโครงการเช่า Infrastructure ระยะเวลา 60 เดือนของ กยศ. ซึ่งยังอยู่ระหว่างการจัดซื้อจัดจ้าง สามารถปรับเปลี่ยนระยะเวลาเช่าเหลือ 12 เดือนได้หรือไม่ โดยไม่ต้องยกเลิกโครงการและขอใหม่ คำตอบคือ มีความเป็นไปได้ แต่ต้องพิจารณาอย่างรอบคอบตามกฎหมายและระเบียบการจัดซื้อจัดจ้างภาครัฐ

ความเป็นไปได้

การปรับเปลี่ยนโครงการเดิมเป็นทางเลือกที่ต้องการ

การป้องกันความเสี่ยง

ช่วยหลีกเลี่ยงผลกระทบจากนโยบาย Cloud First

ข้อจำกัด

มีข้อจำกัดทางกฎหมายและความเสี่ยงในการตีความ

การปรับเปลี่ยนโครงการเดิมจะช่วยให้สามารถใช้งบประมาณเดิมที่ได้รับอนุมัติแล้ว และหลีกเลี่ยงผลกระทบจากนโยบาย Cloud First ที่จะบังคับใช้กับโครงการใหม่ที่ใช้งบประมาณปี 2569 อย่างไรก็ตาม การปรับเปลี่ยนนี้อาจมีข้อจำกัดทางกฎหมาย และมีความเสี่ยงที่จะถูกตีความว่าเป็นโครงการใหม่ ซึ่งจะทำให้ต้องปฏิบัติตามนโยบาย Cloud First ทันที

1. สรุปประเด็นปัญหาและเป้าหมาย

1.1 สภาพปัจจุบันของโครงการ

โครงการเช่า Infrastructure ระยะเวลา 60 เดือน

โครงการเช่า Infrastructure ของกองทุนเงินให้กู้ยืมเพื่อการศึกษา (กยศ.) มีระยะเวลาเช่าทั้งสิ้น 60 เดือน หรือเท่ากับ 5 ปี มีการประกาศรับฟังความคิดเห็นเกี่ยวกับร่างขอบเขตของงาน (TOR) ไปแล้ว [559]

ขอบเขตโครงการ

ครอบคลุมการเช่าเฉพาะ Hardware (Server, Storage, Network Equipment) และ Software แต่ไม่รวมถึง Cloud native Services

สถานะโครงการ

อยู่ในสถานะ "ระหว่างการจัดซื้อจัดจ้าง" มีการจัดพิธีลงนามข้อตกลงคุณธรรมแล้วเมื่อวันที่ 25 มีนาคม 2025 [560]

ประเภทงบประมาณ

ใช้งบประมาณประเภท "งบประมาณรายจ่ายประจำปี"

1.2 อุปสรรคสำคัญ: นโยบาย Cloud First

ผลกระทบจากการยกเลิกโครงการและขอใหม่

หาก กยศ. ตัดสินใจยกเลิกโครงการเดิมและขอใหม่ จะต้องใช้งบประมาณของปีงบประมาณ 2569 ซึ่งจะต้องอยู่ภายใต้นโยบาย Cloud First

คณะกรรมการนโยบาย Cloud First Policy ได้มีมติชัดเจนว่าหน่วยงานภาครัฐจะต้องจัดซื้อบริการคลาวด์ผ่านกลไกกลางเท่านั้น [558]

ความจำเป็นในการเช่า GDCC

หากใช้งบประมาณปี 2569 จะต้องพิจารณาเช่า Government Data Center and Cloud (GDCC) ซึ่งเป็นศูนย์กลางคลาวด์ของรัฐบาล [589]

1.3 คำถามสำคัญสำหรับคณะกรรมการ

สามารถปรับเปลี่ยนโครงการเดิมได้หรือไม่?

การปรับลดระยะเวลาเช่าจาก 60 เดือนเป็น 12 เดือนโดยไม่ต้องยกเลิกโครงการเดิม

มีทางเลือกอื่นใดเพื่อหลีกเลี่ยงนโยบาย Cloud First?

การขอข้อยกเว้นหรือการออกแบบโครงการใหม่ให้ไม่เข้าข่ายเกณฑ์

ผลกระทบในระยะยาวหลังจาก 12 เดือนแรกคืออะไร?

การวางแผนสำหรับการเช่าต่อเนื่องในปี 2570 และต่อไป

2. ทางเลือกที่ 1: การปรับเปลี่ยนโครงการเดิม (Modification)

แนวคิด: การปรับลดระยะเวลาเช่าโดยไม่ยกเลิกโครงการ

การปรับเปลี่ยนโครงการเดิมโดยการลดระยะเวลาการเช่าจาก 60 เดือนเหลือ 12 เดือน โดยไม่ต้องยกเลิกโครงการทั้งหมดและเริ่มต้นกระบวนการจัดซื้อจัดจ้างใหม่ เนื่องจากโครงการยังคงอยู่ในขั้นตอนของการจัดซื้อจัดจ้าง

ข้อดีของการปรับเปลี่ยนโครงการเดิม

การใช้งบประมาณเดิมที่ได้รับอนุมัติแล้ว

การใช้งบประมาณที่ได้รับอนุมัติแล้วจะช่วยประหยัดเวลาและทรัพยากร ลดความเสี่ยงจากการไม่ได้รับการอนุมัติงบประมาณใหม่

การหลีกเลี่ยงกระบวนการของบประมาณใหม่

หลีกเลี่ยงกระบวนการที่ซับซ้อนและใช้เวลานาน ทำให้โครงการสามารถดำเนินการได้รวดเร็วขึ้น

การลดความเสี่ยงจากนโยบาย Cloud First

ช่วยให้ กยศ. สามารถดำเนินการตามแผนเดิมได้โดยไม่ต้องเปลี่ยนไปใช้บริการคลาวด์จาก GDCC

ข้อเสียและข้อจำกัด

ข้อจำกัดทางกฎหมาย

การปรับเปลี่ยนโครงการที่อยู่ระหว่างการจัดซื้อจัดจ้างอาจมีข้อจำกัดทางกฎหมาย ต้องศึกษาข้อกำหนดของระเบียบสำนักนายกรัฐมนตรีว่าด้วยการพัสดุ พ.ศ. 2560

ความเป็นไปได้ในการเปลี่ยนแปลงสัญญา

หากลงนามสัญญาแล้ว การปรับเปลี่ยนจะต้องได้รับความยินยอมจากทั้งสองฝ่าย และอาจต้องมีการเจรจาต่อรองใหม่

ความเสี่ยงต่อการถูกตีความว่าเป็นโครงการใหม่

ความเสี่ยงที่สำคัญที่สุดคือการถูกตีความว่าการปรับเปลี่ยนเป็นโครงการใหม่ ซึ่งจะต้องอยู่ภายใต้นโยบาย Cloud First

ข้อมูลสนับสนุนจากการวิจัย

การวิเคราะห์กรณีศึกษาจากประเทศอื่น ๆ

การศึกษากรณีศึกษาจากประเทศที่มีนโยบาย Cloud First คล้ายคลึงกัน เพื่อเข้าใจแนวทางปฏิบัติและข้อยกเว้นที่เป็นไปได้

การศึกษาข้อกำหนดทางกฎหมายของไทย

ศึกษาข้อกำหนดทางกฎหมายของไทยเกี่ยวกับการจัดซื้อจัดจ้างภาครัฐ เพื่อให้เข้าใจขอบเขตในการปรับเปลี่ยนโครงการ

3. ทางเลือกที่ 2: การยกเลิกโครงการเดิมและขอใหม่ (Cancellation & Re-procurement)

แนวคิด: การเริ่มต้นกระบวนการจัดซื้อจัดจ้างใหม่ทั้งหมด

การยกเลิกโครงการเช่า Infrastructure 60 เดือนเดิมและเริ่มต้นกระบวนการจัดซื้อจัดจ้างใหม่ทั้งหมดสำหรับโครงการเช่า 12 เดือน โดยออกแบบโครงการใหม่ให้เหมาะสมกับสถานการณ์ปัจจุบัน

ข้อดีของการยกเลิกและขอใหม่

การออกแบบโครงการใหม่ให้สอดคล้องกับความต้องการปัจจุบัน

สามารถออกแบบโครงการใหม่ให้มีระยะเวลาเช่า 12 เดือน และมีขอบเขตที่เหมาะสมกับความต้องการของหน่วยงาน

การเปิดโอกาสให้ผู้ให้บริการรายใหม่เข้าร่วมประมูล

เปิดโอกาสให้ผู้ให้บริการรายใหม่ ๆ เข้าร่วมประมูล ซึ่งอาจช่วยให้ได้รับข้อเสนอที่ดีกว่า

ข้อเสียและข้อจำกัด

การใช้งบประมาณปี 2569 ซึ่งอยู่ภายใต้นโยบาย Cloud First

จะต้องใช้งบประมาณปี 2569 ซึ่งจะทำให้โครงการใหม่ต้องอยู่ภายใต้นโยบาย Cloud First อย่างเคร่งครัด

ความจำเป็นในการเช่า GDCC

มีความเป็นไปได้สูงที่จะต้องพิจารณาเช่า Government Data Center and Cloud (GDCC) ซึ่งอาจมีข้อจำกัด

การสูญเสียเวลาและทรัพยากร

สูญเสียเวลาและทรัพยากรที่ใช้ในการดำเนินการมาแล้ว รวมถึงความล่าช้าในการดำเนินโครงการ

ผลกระทบจากนโยบาย Cloud First

การบังคับใช้นโยบาย

โครงการใหม่จะต้องปฏิบัติตามนโยบาย Cloud First อย่างเคร่งครัด

ข้อจำกัดในการเลือก

จำกัดความยืดหยุ่นในการเลือกผู้ให้บริการที่ไม่ใช่ GDCC

ผลกระทบต่อการออกแบบ

มีข้อจำกัดในความยืดหยุ่นในการออกแบบระบบให้ตรงกับความต้องการ

4. ทางเลือกที่ 3: การขอข้อยกเว้นจากนโยบาย Cloud First (Exemption)

แนวคิด: การยื่นขอให้โครงการไม่อยู่ภายใต้ข้อบังคับของนโยบาย

การขอข้อยกเว้นจากนโยบาย Cloud First โดยอ้างเหตุผลว่าโครงการของ กยศ. มีลักษณะเฉพาะที่ไม่สามารถใช้บริการคลาวด์ได้ หรือการใช้บริการคลาวด์ไม่ตรงกับความต้องการของโครงการ

เหตุผลสำหรับการขอข้อยกเว้น

ลักษณะของโครงการ: เช่าเฉพาะ Hardware และ Software

โครงการเช่าเฉพาะ Hardware และ Software แบบดั้งเดิม ไม่ใช่การใช้บริการคลาวด์ ซึ่งแตกต่างจากโมเดลบริการคลาวด์ที่มีค่าใช้จ่ายแบบ Pay-as-you-go

การไม่รวม Cloud Native Services

การไม่รวม Cloud Native Services ในโครงการเป็นข้อโต้แย้งที่สำคัญ เนื่องจากนโยบาย Cloud First มุ่งเน้นการส่งเสริมการใช้งานบริการคลาวด์เป็นหลัก

การใช้งบประมาณประจำปี

การใช้งบประมาณประจำปีสำหรับโครงการเช่าระยะยาว 60 เดือนอาจสร้างความซับซ้อนในการบริหารจัดการ

กระบวนการและขั้นตอนในการขอข้อยกเว้น

การเตรียมเอกสารและข้อมูลสนับสนุน

เตรียมเอกสารและข้อมูลสนับสนุนอย่างรอบคอบ รวมถึงรายละเอียดโครงการ เหตุผลที่ไม่สามารถใช้บริการคลาวด์ได้ ข้อมูลการวิจัยและกรณีศึกษา

การยื่นขอต่อหน่วยงานที่เกี่ยวข้อง

ยื่นขอต่อหน่วยงานที่เกี่ยวข้อง เช่น สำนักงานพัฒนารัฐบาลดิจิทัล (DGA) ตามขั้นตอนและระเบียบที่กำหนด

การชี้แจงเหตุผลต่อคณะกรรมการ

ชี้แจงเหตุผลต่อคณะกรรมการที่เกี่ยวข้องอย่างชัดเจน มีเหตุผล และมีข้อมูลสนับสนุน

ความเป็นไปได้ในการได้รับข้อยกเว้น

การวิเคราะห์จากนโยบายและแนวทางปฏิบัติ

วิเคราะห์จากนโยบายและแนวทางปฏิบัติของนโยบาย Cloud First เพื่อเข้าใจความเป็นไปได้ในการได้รับข้อยกเว้น

การศึกษากรณีศึกษาที่คล้ายคลึงกัน

ศึกษากรณีศึกษาจากหน่วยงานอื่น ๆ ที่เคยขอข้อยกเว้น เพื่อเข้าใจแนวทางปฏิบัติและโอกาสในการได้รับข้อยกเว้น

5. การวิเคราะห์ผลกระทบในระยะยาว

5.1 สถานการณ์หลังจาก 12 เดือนแรก

กรณีปรับเปลี่ยนโครงการเดิม

หากโครงการเดิมถูกปรับเปลี่ยนให้เหลือ 12 เดือน กยศ. อาจต้องพิจารณาว่าจะเช่าต่อหรือไม่ หากต้องเช่าต่อ ก็อาจต้องเริ่มกระบวนการจัดซื้อจัดจ้างใหม่

กรณียกเลิกและขอใหม่

หากโครงการเดิมถูกยกเลิกและขอใหม่ กยศ. อาจต้องพิจารณาว่าจะเช่าต่อจาก GDCC หรือไม่ ซึ่งอาจมีข้อจำกัด

กยศ. อาจต้องเปลี่ยนไปใช้บริการคลาวด์จาก GDCC ในระยะยาว

6. ข้อเสนอแนะและคำแนะนำสำหรับคณะกรรมการ

สรุปข้อดีและข้อเสียของแต่ละทางเลือก

ทางเลือก ข้อดี ข้อเสีย
การปรับเปลี่ยนโครงการเดิม • ใช้งบประมาณเดิมที่ได้รับอนุมัติแล้ว
• หลีกเลี่ยงกระบวนการของบประมาณใหม่
• ลดความเสี่ยงจากนโยบาย Cloud First 		• มีข้อจำกัดทางกฎหมาย
• ความเสี่ยงต่อการถูกตีความว่าเป็นโครงการใหม่
การยกเลิกและขอใหม่ • ออกแบบโครงการใหม่ให้สอดคล้องกับความต้องการปัจจุบัน
• เปิดโอกาสให้ผู้ให้บริการรายใหม่เข้าร่วมประมูล 		• ใช้งบประมาณปี 2569 ซึ่งอยู่ภายใต้นโยบาย Cloud First
• ต้องเช่า GDCC
• สูญเสียเวลาและทรัพยากร
การขอข้อยกเว้น • หลีกเลี่ยงนโยบาย Cloud First
• รักษาโครงการเดิมไว้ 		• ไม่แน่นอนว่าจะได้รับข้อยกเว้นหรือไม่
• ต้องเตรียมเอกสารและข้อมูลสนับสนุนอย่างรอบคอบ

การจัดลำดับความสำคัญของปัจจัยต่าง ๆ

1 การหลีกเลี่ยงนโยบาย Cloud First

เป็นปัจจัยสำคัญที่สุด เนื่องจากนโยบายนี้จะบังคับให้ต้องเช่า GDCC ซึ่งอาจไม่ตรงกับความต้องการเฉพาะทางของโครงการ

2 การใช้งบประมาณเดิม

เป็นปัจจัยสำคัญรองลงมา ช่วยประหยัดเวลาและทรัพยากร ลดความเสี่ยงจากการไม่ได้รับการอนุมัติงบประมาณใหม่

3 ความยืดหยุ่นในการออกแบบระบบ

ความสำคัญในการปรับแต่งระบบให้ตรงกับความต้องการเฉพาะทาง ซึ่งอาจไม่สามารถทำได้หากต้องใช้บริการจาก GDCC

ข้อเสนอแนะเชิงกลยุทธ์

การผสมผสานทางเลือกต่าง ๆ

พิจารณาการผสมผสานทางเลือกต่าง ๆ เพื่อให้ได้ผลลัพธ์ที่ดีที่สุด เช่น

การปรับเปลี่ยนโครงการเดิมให้เหลือ 12 เดือน และในเวลาเดียวกันก็เตรียมการขอข้อยกเว้นจากนโยบาย Cloud First สำหรับโครงการในอนาคต

การเตรียมแผนสำรอง (Plan B)

เตรียมแผนสำรองเพื่อให้มั่นใจว่าหน่วยงานสามารถดำเนินงานได้อย่างต่อเนื่อง รวมถึงการศึกษาความเป็นไปได้ในการใช้บริการจาก GDCC และการวางแผนการย้ายระบบ

ขั้นตอนและแผนการดำเนินงานต่อไป

การรวบรวมข้อมูลและเอกสาร

รวบรวมข้อมูลทางกฎหมาย ข้อมูลการวิจัย กรณีศึกษา และข้อมูลเกี่ยวกับนโยบาย Cloud First และ GDCC

ระยะเวลา: 1-2 สัปดาห์

การประสานงานกับหน่วยงานที่เกี่ยวข้อง

ประสานงานกับหน่วยงาน เช่น DGA และ GDCC เพื่อให้เข้าใจถึงนโยบายและข้อกำหนดต่าง ๆ อย่างชัดเจน

ระยะเวลา: 2-3 สัปดาห์

การกำหนดกรอบเวลาในการตัดสินใจ

กำหนดกรอบเวลาที่ชัดเจนสำหรับการรวบรวมข้อมูล ประสานงาน และการพิจารณาของคณะกรรมการ

ระยะเวลา: 4-6 สัปดาห์

ข้อเสนอแนะเพิ่มเติม

คณะกรรมการควรพิจารณาทางเลือกที่ 1 (การปรับเปลี่ยนโครงการเดิม) เป็นทางเลือกหลัก เนื่องจากมีความเสี่ยงต่ำที่สุดและสามารถใช้งบประมาณเดิมได้ แต่ควรเตรียมแผนสำรองในกรณีที่ไม่สามารถดำเนินการได้

สรุปผลการวิเคราะห์

จากการวิเคราะห์อย่างรอบคอบ การปรับเปลี่ยนโครงการเดิมเป็นทางเลือกที่เหมาะสมที่สุดสำหรับ กยศ. ภายใต้ข้อจำกัดและความเสี่ยงที่มีอยู่ ทั้งนี้ควรดำเนินการอย่างรอบคอบและมีแผนสำรองเพื่อรองรับสถานการณ์ที่อาจเกิดขึ้นในอนาคต

การพิจารณาอย่างรอบคอบ

ต้องพิจารณาทุกปัจจัยรวมถึงกฎหมาย ระเบียบ และนโยบายที่เกี่ยวข้อง

การบริหารความเสี่ยง

ต้องมีแผนรองรับสำหรับสถานการณ์ต่าง ๆ ที่อาจเกิดขึ้นในอนาคต

การประสานงานที่ดี

ต้องมีการประสานงานกับหน่วยงานที่เกี่ยวข้องอย่างใกล้ชิด

การวิเคราะห์นี้จัดทำขึ้นเพื่อประกอบการพิจารณาของคณะกรรมการ
